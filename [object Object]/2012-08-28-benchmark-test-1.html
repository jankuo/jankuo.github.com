基准测试(Benchmark Test, 即BMT)

###基准定义：

基准测试是指通过设计科学的测试方法、测试工具和测试系统，实现对一类测试对象的某项性能指标进行定量的和可对比的测试。

###目前认知：

  1. 语言之间：基准测试成为很多WEB初学者挑选语言的重要参考依据，即运行同样的程序得出时间的快慢而比较语言的速度。

  2. 语言内部：比较不同框架之间运行同样代码片段 比较框架之间的并发率。

###新认知：

  1. 作为框架优化参考依据

  2. 作为程序优化参考依据

REQUEST PER SECONDS 是 ab 测试中重要一项指标。

当框架使用MVC 模式运行 helloworld 程序时，得出的每秒并发数量 是框架优化的起点，是程序优化的终点。
而正常情况下请求一个资源，得出的每秒并发连接数 是程序优化的起点。

我们可以使用曲线 BMT: y=1000n/x (x>0,n表示单位秒) 来记录一序列优化后结果：

曲线BMT的斜率的绝对值k 表示我们进一步优化的难度
曲线BMT与 y=-x 的交点O(x,y)是我们基准测试的值。
而曲线BMT在点O(x,y)点的斜率 k^o是性能调优的转折点.

斜率越大，性能调优带来的效果就越不明显，而调优的难度也就越大

说得有点绕口了，

以 thinkphp 为例，这个框架的helloworld的 REQUEST PER SECONDS 是 167
那么就有：

n=1s
y=167
x=5.98ms

即交点O1(5.98,167);

而 thinkphp官网 首页的 REQUEST PER SECONDS 为 43，那么就有：

n=1s
y=43
x=23ms

即交点O2(23,43);

从O2到O1的斜率变化表示了我们队 thinkphp官网 优化所带来的难度和效率
