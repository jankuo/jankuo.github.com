百度之星程序：du熊学斐波那契

发表于 2017-06-20 | 分类于 技术 |

###Problem Description

du熊对数学一直都非常感兴趣。最近在学习斐波那契数列的它，向你展示了一个数字串，它称之为“斐波那契”串：

11235813471123581347112358........

聪明的你当然一眼就看出了这个串是这么构造的：

  1. 先写下两位在0~9范围内的数字a, b，构成串ab；
  2. 取串最后的两位数字相加，将和写在串的最后面。

上面du熊向你展示的串就是取 a = b = 1 构造出来的串。

显然，步骤1之后不停地进行步骤2，数字串可以无限扩展。现在，du熊希望知道串的第n位是什么数字。

###Input

输入数据的第一行为一个整数T（1 <= T <= 1000）, 表示有T组测试数据；

每组测试数据为三个正整数a, b, n（0 <= a, b < 10, 0 < n <= 10^9）。

###Output

对于每组测试数据，输出一行“Case #c: ans”（不包含引号）

c是测试数据的组数，从1开始。

###Sample Input

3

1 1 2

1 1 8

1 4 8

###Sample Output

Case #1: 1

Case #2: 3

Case #3: 9

###Hint

对于第一、二组数据，串为112358134711235……

对于第三组数据，串为14591459145914……

###偶的 c++ 答案:

#include <stdio.h>
#include <assert.h>

#ifndef DU_FIB_H
#define DU_FIB_H

int du_fib ( int , int, int );
int du_fib_iter ( int, int, int );

#endif

int main( void ) {

    int n, count, a, b , i = 1;

    scanf( "%d",&n );
    while ( i <= n ) {
        scanf( "%d%d%d", &a, &b, &count );
        assert ( count > 0 );

        b = du_fib( a, b , count);

        printf ("Case #%d: %d", i, b);
        if ( i !=  n ) {
            printf("\n");
        }
        i++;
    }


    return 0;
}

int du_fib( int a, int b, int count ) {

    switch ( count ) {
    case 1 :
        return a;
    case 2 :
        return b;
    default:
        return du_fib_iter ( a, b, count-2 );
    }
}

int du_fib_iter( int a, int b, int count ) {
    while ( count > 0) {
        b = a + b;
        a = b - a;

        if ( b > 10 ) {

            a = b / 10;
            b = b % 10;

            if ( count == 1) {
                return a;
            } else if ( count == 2 ) {
                return b;
            }
            count--;
        }
        count--;
    }
    return b;
}

##百鸡问题

##文言文

今有鸡翁一，值钱伍；鸡母一，值钱三；鸡鶵三，值钱一。凡百钱买鸡百只，问鸡翁、母、鶵各几何？答曰：鸡翁四，值钱二十；鸡母十八，值钱五十四；鸡鶵七十八，值钱二十六。又答：鸡翁八，值钱四十；鸡 母十一，值钱三十三，鸡鶵八十一，值钱二十七。又答：鸡翁十二，值钱六十；鸡母四、值钱十二；鸡鶵八十 四，值钱二十八。”

现代语

公鸡每只值5 文钱，母鸡每只值3 文钱，而3 只小鸡值1 文钱。现在用100 文钱买100 只鸡，问：这100 只鸡中，公鸡、母鸡和小鸡各有多少只？

答案是

<公鸡0只，母鸡25只,小鸡75只>
<公鸡4只，母鸡18只,小鸡78只>
<公鸡8只，母鸡11只,小鸡81只>
<公鸡12只，母鸡4只,小鸡84只>

扩展阅读

本问题记载于中国古代约5—6世纪成书的 《张邱建算经》 中，是原书卷下第 38 题，也是全书的最后一题,该问题引出 三元不定方程组，其重要之处在于开创“一问多答”的先例，这是过去中国古算书中所没有的:

  1. 中国古算书的著名校勘者甄鸾和李淳风注释该书时都没给出解法
  2. 约6世纪的算学家谢察微记述过一种不甚正确的解法
  3. 1815年骆腾风使用大衍求一术解决了百鸡问题。
  4. 1874年丁取忠创用一个简易的算术解法 x

##代码[golang]

百钱买百鸡，那么公鸡，母鸡，小鸡，最多100只，最少0只，于是乎:

package main

import "fmt"

func main() {
    var x,y,z int

    count := 0;
    fmt.Printf("第 1 次穷解法:\n")
    for x = 0; x <= 100; x++ {
        for y = 0; y<=100; y++ {
            for z = 0;z <100; z++ {
                if (z%3 == 0) && (x + y + z == 100) && (5*x+3*y+z/3 == 100 )  {
                    fmt.Printf("<公鸡%d只，母鸡%d只,小鸡%d只>\n",x,y,z)
                }
                count++
            }
        }
    }
    fmt.Printf("\n枚举了:%d次\n\n",count)
}

package main

import "fmt"

type Chicken struct {
    X,Y,Z int
}

func (self *Chicken) String() string {
     return fmt.Sprintf("<公鸡:%-3d 只，母鸡:%-3d 只,小鸡:%-3d 只>",self.X,self.Y,self.Z)
}

func main() {         

    var chickens []*Chicken = []*Chicken{
        &Chicken{0,25,75},
        &Chicken{4,18,78},
        &Chicken{8,11,81},
        &Chicken{12,4,84},
    }

    fmt.Printf("第 9 次,既然知道答案，那就一次全部输出吧:\n")
    for _, v := range chickens {
        fmt.Println(v);
    }         
}

总结

  1. 我们大学学习的过程是慢慢的从第一步到第九步的，这段时间，我们学到多少不重要，重要的是学会学习方法和解题方法(如果能用到实战中，那么最好不过了)

  2. 我们实际项目中一般都是遵行 8 -> 1 的过程，也就是说不到万不得已，绝对不会用最原始的 穷举法

  3. 第9次，往往是我们已经知道精确答案，而且知道答案永久不会改变才这么做的。

##描述##

有一个已经排序的数组（升序），数组中可能有正数、负数或0，求数组中元素的绝对值最小的数，要求，不能用顺序比较的方法（复杂度需要小于O（n）），可以使用任何语言实现

例如，数组 {-20，-13，-4, 6, 77,200} ，绝对值最小的是 -4。

##思路##

既然有序，而且时间复杂度小于 O（n）, 第一个想到的就是二分查找！不多说，直接上代码

##代码##

int abs_min ( int myints[], int len ) {

  int left = 0,right = len - 1,middle = 0;

  if ( myints[right] <= 0 ) {
      return myints[right];
  } else if ( 0 <=  myints[left]) {
      return myints[left];
  } else {
        while ( myints[left] < 0 || myints[right] > 0 ) {

            middle = ( left + right ) / 2;
            if ( myints[middle] > 0 ) {
                if ( myints[middle - 1] >= 0 ) {
                    right = middle-1;
                    continue;
               } 
          }
          if ( myints[middle] < 0 ) {
              if ( myints[middle+1] <= 0) {
                      left = middle+1; 
                      continue; 
              }
          }
          left = right = middle;
          break;

        }

        if ( (myints[left] + myints[right]) >= 0) {
            return myints[left];
        }
        return myints[right];
  }
}

完美因数C代码

发表于 2017-06-20 | 分类于 技术 |

##描述##

何一个自然数的约数中都有1和它本身，我们把小于它本身的因数叫做这个自然数的真约数。如6的所有真约数是1、2、3，而且6=1＋2＋3。像这样，一个数所有真约数的和正好等于这个数，通常把这个数叫做完美数。 古希腊人非常重视完美数。毕达哥拉斯发现它之后，人们就开始了对完美数的研究。也许完美数太少了，一直到现在，数学家才发现了29个完美数，而且都是偶完美数。前5个完美数分别是：

6，28，496，8128，33550336

##代码##

#include <stdio.h>

int is_perfect ( int n );

int main ( void )
{
    int i = 2;
    for ( ; i < 10000 ; i++ ) 
    {
        if ( is_perfect ( i ) )
        {
            printf( "%d ", i );
        }
    }
    printf("\n");
    return 0;
}

int is_perfect ( int n )
{
    int min = 2, max = n,sum = 1;
    while ( min <= max) {
        max = n / min;
        if ( (n % min ) == 0 ) {
            sum += min;
            sum += max;
            if ( min == max ) {
                break;
            }
        }
        min++;
    }
    return sum == n;
}

易语言数学难题

发表于 2017-06-20 | 分类于 技术 |

##描述##

用1，2，3，4，5，6，7，8.9 九个数字组成三个三位数（每个数字只能使用一次），是第二个数字是第一个数字的2倍，第三个数字是第二个数字的3被

##答案##

192 384 576 
219 438 657 
273 546 819 
327 654 981

##具体代码##

//数组方式
#include <stdio.h>

void reset_num (int set[]) {
    int i = 1;
    for (; i < 10; i++ ) {
        set[i] = 0;
    }
}

int check ( int num, int set[] ) {
    int tmp = 0,i = 3;
    while ( i > 0 ) {
        tmp = num % 10;
        if ( set[tmp] == 1 ) {
            return 0;
        }
        set[tmp] = 1;

        num = num / 10;
        i--;
    }
    return 1;
}

int main ( void )
{
    int min = 145,max = 329;
    int set[10] = {1,0};
    while ( min <= max ) {
        reset_num ( set );
        if ( check ( min, set ) && check ( min*2, set ) && check ( min*3, set ))
        {
            printf("%d %d %d \n", min, min*2, min*3 );
        }
        min++;
    }

    printf("\n");
    return 0;
}

//位运算方式
#include <stdio.h>

int check ( long num ) {
    int tmp = 0,i = 9, set = 1;
    while ( i > 0 ) {
        i--;
        tmp = num % 10;     
        if ( ( set & ( 1 << tmp ) )  >= 1)  {
            return 0;
        }
        set |= (1 << tmp );
        num = num / 10;
    }

    return 1;
}

int main ( void )
{
    int min = 145,max = 329;
    long tmp = 0;
    while ( min <= max ) {
        tmp = min * 1000000 + min * 2000 + min*3;

        if ( check ( tmp ) )
        {
            printf("%d %d %d \n", min, min*2, min*3 );
        }
        min++;
    }

    printf("\n");
    return 0;
}

克拉克三法则

发表于 2017-06-20 | 分类于 技术 |

##克拉克基本定律##

克拉克基本定律是英国著名科幻作家亚瑟·查理斯·克拉克积累有关科学文化方面的经验提出的。

  1. 如果一个年高德劭的杰出科学家说，某件事情是可能的，那他可能是正确的；
    但如果他说，某件事情是不可能的，那他也许是非常错误的；

  2. 要发现某件事情是否可能的界限，唯一的途径是跨越这个界限，从不可能跑到可能中去；

  3. 任何非常先进的技术，初看都与魔法无异。
